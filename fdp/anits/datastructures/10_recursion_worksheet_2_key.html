The Art of Recursion

Outputs and brief traces for every question. Use the notes to highlight what each program is teaching about recursion.

I.
Part One

Warm-up — Plain functions

Three short programs without recursion. The goal is simply to confirm students can trace.

01 Reverse the digits of a number. 
def reverse_num(n):
    rev = 0
    while n > 0:
        rev = rev * 10 + n % 10
        n = n // 10
    return rev

print(reverse_num(1234))
Output
4321
Trace. rev=0, n=1234 → rev=4, n=123 → rev=43, n=12 → rev=432, n=1 → rev=4321, n=0. The least-significant digit gets pushed in first and ends up most-significant — a useful idea to remember when you see digit problems again recursively.
02 Multi-branch decision. 
def grade(marks):
    if marks >= 90:
        return "A"
    elif marks >= 75:
        return "B"
    elif marks >= 60:
        return "C"
    else:
        return "F"

print(grade(82))
print(grade(58))
print(grade(95))
Output
B F A
Trace. 82: not ≥ 90, but ≥ 75 → B.   58: fails all checks → F.   95: ≥ 90 → A. The first matching branch wins — exactly how a base-case ladder will work in recursion.
03 A function calling another function. 
def cube(x):
    return x * x * x

def diff(a, b):
    return cube(a) - cube(b)

print(diff(3, 2))
Output
19
Trace. cube(3) = 27, cube(2) = 8 → 27 − 8 = 19. Each call to cube opens its own little world, returns a value, and disappears — the same mental model students will need for every recursive call.

II.
Part Two

Recursion — Trace the output

Each function calls itself, but the way it shrinks the problem is different every time.

01 Step size of 2. 
def fun(n):
    if n <= 0:
        return
    print(n, end=" ")
    fun(n - 2)

fun(7)
Output
7 5 3 1
Note. The recursive step does not have to be n - 1. Any step that guarantees the base case is reached will do. The base case is also subtle here — n <= 0 instead of n == 0 — because fun(1) would otherwise call fun(-1) and miss n == 0.
02 Filtered print on the way up. 
def fun(n):
    if n == 0:
        return
    fun(n - 1)
    if n % 2 == 0:
        print(n, end=" ")

fun(6)
Output
2 4 6
Note. Two ideas combined: print after recurse means values come out smallest-first, and the if n % 2 == 0 filter only lets even values through. Frames for 1, 3, 5 still get pushed and popped — they just decide not to print.
03 Counting digits — but with division, not subtraction. 
def count(n):
    if n == 0:
        return 0
    return 1 + count(n // 10)

print(count(78912))
Output
5
Trace. 78912 → 7891 → 789 → 78 → 7 → 0. Five chops, so the result is 5. The shrinking step is n // 10 — a different rhythm from n - 1, but the same recursive idea.
04 Sum the digits. 
def digit_sum(n):
    if n == 0:
        return 0
    return (n % 10) + digit_sum(n // 10)

print(digit_sum(1234))
Output
10
Trace. digit_sum(1234) = 4 + digit_sum(123) = 4 + 3 + digit_sum(12) = … = 4 + 3 + 2 + 1 + 0 = 10. Same shrinking pattern as Q3, but now the work each frame does is add the last digit instead of just adding 1.
05 Multiplication via repeated addition. 
def mult(a, b):
    if b == 0:
        return 0
    return a + mult(a, b - 1)

print(mult(5, 4))
Output
20
Trace. mult(5,4) = 5 + mult(5,3) = 5 + 5 + mult(5,2) = … = 5 + 5 + 5 + 5 + 0 = 20. Only the second argument shrinks; the first stays fixed. That is enough — recursion only needs one thing to march toward the base case.
06 Two recursive calls with a print between. 
def fun(n):
    if n == 0:
        return
    fun(n - 1)
    print(n, end=" ")
    fun(n - 1)

fun(3)
Output
1 2 1 3 1 2 1
The big idea. This is exactly an in-order traversal of a complete binary tree: visit left subtree, print this node, visit right subtree. Reading the output from left to right is the same as reading the tree's nodes in symmetric order.
This sets up step 4 of the lesson plan beautifully — students have already seen a tree being traversed by recursion before they meet the word "tree".
07 Greatest common divisor — the Euclidean algorithm. 
def gcd(a, b):
    if b == 0:
        return a
    return gcd(b, a % b)

print(gcd(48, 18))
Output
6
Trace. gcd(48, 18) → gcd(18, 12) → gcd(12, 6) → gcd(6, 0) → returns 6.
Both arguments change every call — and they don't shrink uniformly. This is the first program that feels like a real algorithm students might see in a textbook, and it shows recursion can be far more flexible than just n - 1.
