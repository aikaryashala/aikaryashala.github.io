Quick Reference

How Each Language Runs

Each language takes a different path from source code to execution.

C++

Compiled to Native

Source → Machine code. No runtime needed.

1
Source (.cpp) Write the program in C++
2
Compiler (g++ / clang++) Translates to machine code & links libraries
3
Native Executable Runs directly on the CPU — no middleman

Fastest execution · Platform-specific binary

Java

Bytecode + JVM

Source → Bytecode → JIT → Native at runtime.

1
Source (.java) Write the program in Java
2
javac Compiler Compiles to platform-neutral bytecode (.class)
3
JVM + JIT Compiler Interprets bytecode and JIT-compiles hot paths to native

Write once, run anywhere · JVM required

Python

Interpreted

Source → Bytecode → CPython VM.

1
Source (.py) Write the program in Python
2
CPython Compiler Compiles to bytecode (.pyc) automatically
3
Python VM Executes bytecode line-by-line at runtime

Most flexible & dynamic · Slowest raw speed

Environment Setup

Install Ubuntu on Windows via WSL

Open PowerShell or Command Prompt as Administrator and follow these steps.

1

Update WSL

Make sure your WSL installation is up to date before anything else.

PS>wsl --update
2

List Available Distributions

See all Linux distributions you can install. Look for Ubuntu-24.04 in the list.

PS>wsl --list --online
3

Install Ubuntu 24.04

Either command works — use whichever succeeds on your machine.

PS>wsl --install Ubuntu-24.04
    # or
PS>wsl --install -d Ubuntu-24.04
4

Restart Your Machine

After installation completes, restart Windows to apply the changes before the first launch.

5

Set Username & Password

On first launch, Ubuntu will ask you to create a Unix username and password. The password is not visible while you type — that is normal. Type it carefully and press Enter.

⚠️ The cursor will not move and nothing will be shown while typing your password. This is expected.
6

Verify the Installation

Run these commands to confirm WSL and Ubuntu are set up correctly.

PS>wsl --version  # check WSL version
PS>wsl --list --verbose  # see running distros & WSL version per distro

You should see Ubuntu-24.04 listed with state Stopped or Running and version 2.

Session Commands

Linux & C Commands

Commands used during the session — run these inside your Ubuntu terminal.

File System

ls List all files and folders in the current directory
pwd Print Working Directory — shows your current location in the file system
clear Clear the terminal screen

Working with Files

touch sum.c Create a new empty file named sum.c
cat sum.c Print the contents of sum.c to the terminal

Compiling & Running C

clang sum.c Compile sum.c using the Clang compiler
./a.out Run the compiled program — default output file is always a.out
rm a.out Delete the a.out file
clang sum.c -o sum Compile and name the output sum instead of a.out
./sum Run the named executable — sum.c executes
When you run clang sum.c without -o, the compiler always creates a default output file called a.out (short for assembler output). Use -o <name> to give it a meaningful name.

Compilation Stages — Save Intermediate Files

clang --save-temps sum.c -o sum Compile and save all intermediate files generated during compilation
ls List the directory — you will see all the files created
After running --save-temps, the following files are generated:
sum
Final executable
sum.i
Preprocessed source
sum.bc
LLVM bitcode
sum.s
Assembly code
sum.o
Object file
sum.c
Original source

bash file Execute a bash script file directly
