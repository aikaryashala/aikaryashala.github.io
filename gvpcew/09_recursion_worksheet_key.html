Solutions · No. 01 · Recursion

The Art of Recursion

Outputs and brief traces for every question. Use the notes to highlight what each program is teaching about recursion.

I.
Part One

Warm-up — Plain functions

Three short programs without recursion. The goal here is simply to confirm students can trace.

01 Sum of digits via a loop. 
def process(n):
    result = 0
    while n > 0:
        result = result + n % 10
        n = n // 10
    return result

print(process(234))
Output
9
Trace. n=234 → add 4, n becomes 23 → add 3, n becomes 2 → add 2, n becomes 0. Total: 4 + 3 + 2 = 9.
02 Conditional branching. 
def check(a, b):
    if a > b:
        return a - b
    else:
        return b - a + 1

print(check(5, 3))
print(check(2, 7))
Output
2 6
Trace. check(5, 3): 5 > 3 is true → returns 5 − 3 = 2.   check(2, 7): 2 > 7 is false → returns 7 − 2 + 1 = 6.
03 A function calling another function. 
def square(x):
    return x * x

def compute(a, b):
    return square(a) + square(b)

print(compute(3, 4))
Output
25
Trace. square(3) = 9, square(4) = 16 → 9 + 16 = 25. A natural lead-in to recursion: a function calling another function is fine — and a function calling itself is just a special case of that.

II.
Part Two

Recursion — Trace the output

Watch carefully when work happens — before the recursive call, or after.

01 Print, then recurse. 
def fun(n):
    if n == 0:
        return
    print(n, end=" ")
    fun(n - 1)

fun(4)
Output
4 3 2 1
Note. Each call prints before recursing, so values come out in the order they enter — counting down from 4 to 1.
02 Recurse, then print. Only one line moved! 
def fun(n):
    if n == 0:
        return
    fun(n - 1)
    print(n, end=" ")

fun(4)
Output
1 2 3 4
Note. Now the print happens after the recursive call returns. Each call has to wait for the smaller one to finish first, so values come out in reverse order — counting up from 1 to 4. Compare with Q1: position of the print line is everything.
03 Sum of 1 to n. 
def total(n):
    if n == 0:
        return 0
    return n + total(n - 1)

print(total(5))
Output
15
Trace. total(5) = 5 + total(4) = 5 + 4 + total(3) = … = 5 + 4 + 3 + 2 + 1 + 0 = 15. The first recursion that returns a value rather than just printing.
04 Factorial — the textbook example. 
def fact(n):
    if n == 1:
        return 1
    return n * fact(n - 1)

print(fact(5))
Output
120
Trace. fact(5) = 5 × fact(4) = 5 × 4 × fact(3) = … = 5 × 4 × 3 × 2 × 1 = 120.
05 Two prints — one before, one after. 
def fun(n):
    if n == 0:
        return
    print(n, end=" ")
    fun(n - 1)
    print(n, end=" ")

fun(3)
Output
3 2 1 1 2 3
The big idea. Each n is printed twice — once on the way down (before the recursive call) and once on the way up (after it returns). The result is a palindrome:
fun(3): 3 → fun(2): 2 → fun(1): 1 → fun(0): return → 1 → 2 → 3.
This question previews the call stack — each frame is paused on its first print, waits for the children, then resumes for its second print.
06 Power function — recursion with two parameters. 
def power(a, b):
    if b == 0:
        return 1
    return a * power(a, b - 1)

print(power(2, 4))
Output
16
Trace. power(2,4) = 2 × power(2,3) = 2 × 2 × power(2,2) = … = 2 × 2 × 2 × 2 × 1 = 16. Notice that only one argument shrinks each call — that's enough to guarantee the base case is reached.
07 Fibonacci — the function calls itself twice. 
def fib(n):
    if n <= 1:
        return n
    return fib(n - 1) + fib(n - 2)

print(fib(5))
Output
5
Trace. The values build up bottom-up: fib(0)=0, fib(1)=1, fib(2)=1, fib(3)=2, fib(4)=3, fib(5) = fib(4) + fib(3) = 3 + 2 = 5.
This is the first program where each call splits into two children — a perfect setup for explaining the call-stack tree in step 3 of the lesson.
