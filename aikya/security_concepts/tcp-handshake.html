Glossary
SYNSynchronize
"I want to start a conversation. Here's my initial sequence number."
ACKAcknowledge
"I received your byte up to N. Send me the next one."
Sequence Number
A number labelling each byte. Starts random (the ISN) so old packets can't confuse a new connection.
ISNInitial Seq Number
Picked randomly at session start. Different on each side: client (x) and server (y).
Why three packets?
Each side must prove it can both send and receive. Two packets aren't enough — the client wouldn't know its own SYN was received.
