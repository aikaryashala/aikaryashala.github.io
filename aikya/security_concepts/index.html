Reading the wire,

Packet sniffing is the art of capturing the raw conversations flowing across a network — the same bits your browser, OS, and apps quietly exchange every second.

Before you can sniff, you must understand how data moves. Two foundations make everything else possible.

~/lesson — packet-sniffing.intro
$ what is packet sniffing?

A packet sniffer is a program (or device) that intercepts and logs network traffic passing through a network interface. It can read every protocol header — Ethernet, IP, TCP, HTTP — and reveal what is being sent, by whom, to whom, and when.

Sniffers power debugging, performance analysis, security auditing, incident response, and protocol learning. The same techniques used by attackers are used by defenders — understanding both is the goal of this lesson.

01 / Fundamentals

Two things you must know first.

A packet sniffer shows you frames, packets, and segments — the building blocks defined by the OSI model — and conversations that almost always begin with a TCP handshake. Master these two ideas and the output of Wireshark stops looking like noise and starts telling a story.

Foundation 01 Required

The OSI Model — 7 layers of cooperation.

Every captured packet is a stack of headers: Ethernet wraps IP wraps TCP wraps HTTP. A sniffer simply un-stacks them. Without the OSI model, you cannot explain what each header means or where in the stack a problem lives.

The TCP Handshake — how a connection begins.

The SYN → SYN-ACK → ACK exchange is the first thing you see in nearly every TCP capture. Recognising it lets you spot the start of a session, detect failed connections, and understand sequence numbers — the same numbers an attacker manipulates in spoofing and hijacking.

03 / Concept

How does a sniffer see traffic?

Normally, a network card discards any frame that is not addressed to it. A sniffer flips that behaviour by putting the card into promiscuous mode (wired) or monitor mode (Wi-Fi), so it accepts every frame it can hear — then hands the raw bytes to a capture library like libpcap.

From wire to readable packet.

The NIC receives raw bits from the cable or radio. In promiscuous mode it accepts all frames, not just those addressed to your MAC. The OS kernel hands them to libpcap / WinPcap / Npcap, which delivers them to a tool like Wireshark.

Wireshark then walks the layers — Ethernet → IP → TCP → HTTP — exactly the way the OSI model describes, and shows you each header in plain text.

04 / Vocabulary

Words you will see everywhere.

Capture tools assume you know this language. Each term ties back to a layer of the OSI model — knowing which one helps you filter and read traffic with precision.

01 · L2

Frame

A unit of data at the Data Link layer, wrapped in an Ethernet header containing source & destination MAC addresses.

02 · L3

Packet

An IP-layer unit carrying source & destination IP addresses, allowing routing across networks.

03 · L4

Segment

A TCP-layer unit with port numbers, sequence numbers, and flags like SYN, ACK, FIN.

04

Promiscuous Mode

NIC setting that accepts every frame on the wire, not just frames addressed to its own MAC.

05

BPF Filter

Berkeley Packet Filter expressions like tcp port 443 or host 8.8.8.8 that narrow what you capture.

06

PCAP

The standard file format (.pcap, .pcapng) for storing captured traffic so it can be replayed or analysed later.

07

Port Mirroring (SPAN)

A switch feature that copies traffic from one port to another, so a sniffer attached to the mirror port can see all conversations.

08

ARP Spoofing

An active attack that tricks a switch into forwarding traffic to the attacker — used to enable sniffing on a switched LAN.

09

TLS / Encryption

Above TCP, payloads are typically encrypted. A sniffer still sees the metadata (IPs, ports, SNI) but not the message body.

05 / Toolbox

Tools of the trade.

Once the fundamentals click, these are the tools you will use to actually capture and dissect traffic. Start with Wireshark for the visual experience, then graduate to tcpdump on a remote server.

Wireshark Cross-platform GUI analyser. Decodes thousands of protocols, colour-codes conversations, and lets you follow a TCP stream end-to-end. GUI · Cross-platform
tcpdump The classic command-line sniffer for Unix/Linux. Lightweight, scriptable, perfect for capturing on remote servers and writing to .pcap. CLI · *nix
tshark The terminal version of Wireshark. Same powerful dissectors, scriptable output — great for automation pipelines. CLI
Scapy Python library to capture, craft, and replay packets. The bridge from learning to building your own tools. Python · Library
ngrep grep, but for network traffic. Search live captures for strings or regex patterns. Handy for quick text-level inspection. CLI · *nix
!

Use these powers responsibly.

Sniffing traffic on a network you do not own — or do not have explicit permission to monitor — is illegal in most jurisdictions. Practise on your own lab, on virtual machines, or on networks where you have written authorisation. The goal of this lesson is defence and understanding, not intrusion.
